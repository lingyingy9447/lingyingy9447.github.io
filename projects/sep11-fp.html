Platformer Game

Context/Content

The "Freedom Project" for SEP11 is all about applying what students had learned throughout the sep11 course and what they learn from their tool to create anything they want that uses javascript. The purpose of this blog is to show the process of how the students made their freedom project. It shows how the students process everything in detail and the challenges they faced throughout the year.

For my project, I am making a platformer game for which the user has to collect coins and escape from the enemies to go to the next level. This blog will walk through my process of independently studying KaboomJs to make my platformer game. It will show how I used what I learned from class and what I learned from kaboom js, a javascript library, to make my game.

Reflection

One of my biggest challenges is making the “toto” (enemies) move back and forth. Initially, the toto was flowing in the air which looked kind of weird. We try various attempts to make the toto move back and forth between two certain positions. We tried using `.pos` to control the position of the tot but it only worked for the first toto. We couldn’t get the position of every single toto so we ended up making the remaining toto stay in one position. Another challenge I faced was looping the levels. The code we have, `level: (level + 1)’ , works for going from level one to level three but it doesn’t when you want to go back to level one because our game map is array inside of array so the index will start with 0. If we are currently at level 3, the index will be 2 and 2 plus one will give us three but we do not have a level with an index of 3. After using a concrete example, we realized that adding ` % maps.length` will work because 2 mod 2 will give us a remainder of zero which brings us back to level 1.

One of my takeaways from doing this project is that using concrete examples will help you process the code better. Additionally, using console.log will help us because it will show you where the problem occurs in your code. It will be easier for us to debug our code.

My next step for this project is to add an instruction scene because it will make it easier for the player to understand what they need to do. Additionally, I want to add audio to the game so it's more entertaining.
